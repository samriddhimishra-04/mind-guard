### **Part 3: Create Journal Entry - SHOWS ALL 3 LLM CALLS! (8:00 - 13:00)**

**Action**: Click "New Journal Entry" button

**Fill in journal:**
```
Title: "Work stress is overwhelming me"
Content: "I am feeling very anxious and overwhelmed today. Work deadlines are piling up and I feel hopeless about the situation. I cannot sleep well at night. Everything feels so stressful and I worry constantly. I need help."
Mood: ANXIOUS
```

**Click Save** → Watch the backend process the request

**Behind the scenes - 3 LLM API calls execute:**

```
T+0ms:    Request received with journal text
T+50ms:   ┌─ LLM CALL #1 starts
          │  (Sentiment Analysis - DistilBERT)
          │  Input: Journal text
T+100ms:  │
T+150ms:  ├─ LLM CALL #2 starts  
          │  (Distress Detection - Custom Model)
          │  Input: Journal text
T+500ms:  │
          └─ Calls #1 and #2 complete
            Results: Sentiment=0.92, Distress=8.2

T+600ms:  Decision: Distress (8.2) > Threshold (5.0)? YES!
          → TRIGGER LLM CALL #3

T+700ms:  ┌─ LLM CALL #3 starts
          │  (AI Analysis - Text2Text Generation)
          │  Input: Journal + Sentiment + Distress scores
T+1200ms: │  
T+1800ms: └─ Call #3 completes with generated analysis

T+1900ms: Alert created in database
T+1950ms: Response sent to client with all LLM scores
```

**Result shown to patient:**
```
✅ Journal entry created
   • Sentiment Score: 0.92 (very negative)
   • Distress Level: 8.2/10 (critical)
   • Entry ID: [generated]
```

**Explanation to audience:**
> "Notice the response includes TWO pieces of information:
> 
> 1. **Sentiment Score (0.92)** - This came from LLM Call #1 using DistilBERT. It analyzed the emotional tone and returned 0.92 on a scale where 0 is very negative and 1 is very positive. This patient is expressing extremely negative emotions.
>
> 2. **Distress Level (8.2/10)** - This came from LLM Call #2. It detected crisis keywords and severity indicators: 'anxious', 'overwhelmed', 'hopeless', 'cannot sleep', 'need help'. It scored 8.2 out of 10 - critical level.
>
> Because this distress score exceeds our safety threshold of 5.0, the system AUTOMATICALLY triggered LLM Call #3, which is currently generating personalized recommendations based on both the sentiment and distress scores. This shows intelligent decision-making: the LLM outputs drive the business logic."

### **Part 5: Explain the Architecture (18:00 - 20:00)**

**Talking points:**

> "Mind-Guard demonstrates **three LLM models working together** in real-world mental health support:
>
> **The Architecture:**
> 
> When a patient creates a journal entry:
>
> 1. **Sentiment Analysis** (Call #1)
>    - Model: DistilBERT
>    - Purpose: Measure emotional tone
>    - Output: Score from -1 (very negative) to 1 (very positive)
>    - This patient: 0.92 (extremely negative tone)
>
> 2. **Distress Detection** (Call #2)
>    - Model: Custom trained model
>    - Purpose: Identify crisis indicators and severity
>    - Output: Score from 0 (fine) to 10 (severe crisis)
>    - This patient: 8.2/10 (critical level)
>    - **Decision Point**: if distress > 5.0, trigger next call
>
> 3. **AI Analysis Generation** (Call #3)
>    - Model: Text-to-Text generation model
>    - Purpose: Generate personalized recommendations
>    - Input: Combines journal + sentiment score + distress score
>    - Output: Natural language analysis and action items
>    - Time: ~1 second after Calls #1 & #2 complete
>
> **Why This Matters:**
> - All 3 calls complete in ~2 seconds
> - The LLM outputs drive business logic (distress > 5.0 = create alert)
> - The therapist gets intelligent, context-aware alerts
> - The system prioritizes high-risk patients automatically
> - Shows how multiple AI models can be orchestrated in production systems"

## 🧪 Live Testing

You can test the complete flow with this command:

```bash
#!/bin/bash
API="http://localhost:8081"
TS=$(date +%s)

# Register patient
TOKEN=$(curl -s -X POST $API/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test_'$TS'","email":"test'$TS'@demo.com","password":"Test@123456","firstName":"Test","lastName":"User","role":"PATIENT"}' \
  | grep -o '"accessToken":"[^"]*' | cut -d'"' -f4)

# Log mood
curl -s -X POST $API/api/moods \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"mood":"ANXIOUS","intensityLevel":8}' | grep '"id"' && echo "✅ Mood logged"

# Create journal
curl -s -X POST $API/api/journals \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"Test","content":"I feel anxious and hopeless","mood":"ANXIOUS"}' \
  | grep -o '"sentimentScore":[0-9.]*' && echo "✅ Journal created with LLM scores"
```
