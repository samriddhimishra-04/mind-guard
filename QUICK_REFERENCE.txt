QUICK START (3 STEPS)
────────────────────────────────────────────────────────────────────────────
1. DATABASE:
   createdb mindguard
   psql -U postgres -d mindguard -f backend/src/main/resources/scripts/init.sql

2. BACKEND:
   cd backend
   cp .env.example .env
   # Edit .env with values (JWT_SECRET, ENCRYPTION_KEY, HF API Key)
   mvn clean install
   mvn spring-boot:run
   → Runs on: http://localhost:8080/api

3. FRONTEND:
   cd frontend
   npm install
   npm start
   → Opens: http://localhost:4200

SECURITY FEATURES
────────────────────────────────────────────────────────────────────────────
✓ JWT Authentication       (Access + Refresh tokens, 24hr expiration)
✓ Role-Based Access        (PATIENT, THERAPIST, ADMIN)
✓ AES-256 Encryption       (Sensitive data at rest)
✓ SQL Injection Prevention  (JPA parameterized queries)
✓ Input Validation         (Annotation-based @Valid)
✓ Password Hashing         (bcrypt, 12 salt rounds)
✓ CORS Protection          (Restricted origins only)
✓ Audit Logging            (All sensitive operations logged)

DATABASE TABLES (7 Total)
────────────────────────────────────────────────────────────────────────────
1. users                  → Patients, Therapists, Admins (with roles)
2. journal_entries        → Encrypted journal content with mood tracking
3. mood_logs              → Daily mood tracking (1-10 scale)
4. journal_analyses       → AI analysis results and risk scores
5. alerts                 → Emergency escalation notifications
6. alert_responses        → Therapist responses and actions
7. therapist_patients     → Therapist-patient assignments

AI WORKFLOW (Hugging Face - Cloud Based)
────────────────────────────────────────────────────────────────────────────
User Creates Journal Entry
        ↓
[HF Call 1] Sentiment Analysis (distilbert model)
        ↓
[HF Call 2] Distress Detection + Self-Harm Indicators (zero-shot)
        ↓
Risk Score Calculation (0-100 scale)
        ├─ Risk > 70: EMERGENCY
        │  ├─ Create EMERGENCY alert
        │  ├─ [HF Call 3] Generate Therapist Summary
        │  └─ Send real-time notification to assigned therapist
        │
        └─ Risk < 70: Standard
           ├─ Generate wellness guidance
           └─ Store as recommendations

TECHNOLOGY STACK
────────────────────────────────────────────────────────────────────────────
Frontend:        Angular 18 + TypeScript 5.3 + Angular Material
Backend:         Java 17 + Spring Boot 3.3 + Spring Security
Database:        PostgreSQL 14 + HikariCP (connection pooling)
AI:              Hugging Face Inference API (cloud-based, no local models)
Build Tools:     Maven 3.8+ (Backend), NPM (Frontend)
Authentication:  JWT (jjwt 0.12.3)
Encryption:      AES-256 (javax.crypto)

IMPORTANT URLS & ENDPOINTS
────────────────────────────────────────────────────────────────────────────
Development URLs:
  Frontend:     http://localhost:4200
  Backend API:  http://localhost:8080/api

Key API Endpoints:
  POST   /auth/register              → Register new user
  POST   /auth/login                 → Login with credentials
  POST   /auth/refresh               → Refresh access token
  GET    /journals                   → List user's journal entries
  POST   /journals                   → Create new journal entry
  GET    /journals/{id}              → Get specific journal entry
  PUT    /journals/{id}              → Update journal entry
  DELETE /journals/{id}              → Delete journal entry
  GET    /moods                      → Get mood history
  POST   /moods                      → Log new mood
  GET    /moods/summary              → Get mood statistics
  POST   /analysis/analyze           → Analyze journal entry
  GET    /alerts                     → Get alerts (therapist only)
  PUT    /alerts/{id}/acknowledge    → Acknowledge alert
  POST   /alerts/{id}/response       → Add therapist response
