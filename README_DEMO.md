## 📚 The 3 LLM API Calls

### 1️⃣ Sentiment Analysis
```
Model: distilbert-base-uncased-finetuned-sst-2-english
Purpose: Analyze emotional tone of journal entries
Input: "I am feeling very anxious and stressed..."
Output: Sentiment Score (0.0 = negative, 1.0 = positive)
```

### 2️⃣ Distress Level Detection
```
Model: Custom distress detection
Purpose: Identify crisis indicators and mental health concerns
Input: "I am feeling very anxious and overwhelmed..."
Output: Distress Level (0-10 scale)
```

### 3️⃣ AI Analysis Generation
```
Model: Text generation (Hugging Face)
Purpose: Generate personalized insights and recommendations
Input: Journal text + Sentiment + Distress scores
Output: AI-generated support strategies
```

## 🚀 Quick Start (20 Minutes)

### Prerequisites
- ✅ Backend JAR compiled (mindguard-1.0.0.jar)
- ✅ Frontend built (npm install done)
- ✅ Hugging Face API key: `<YOUR_HUGGING_FACE_API_KEY>`

### 1. Start Backend (2 minutes)
```bash
cd c:\JAVA\mind-guard\backend

java -jar target/mindguard-1.0.0.jar \
  --server.port=8081 \
  --huggingface.api-key=<YOUR_HUGGING_FACE_API_KEY>
```

**Wait for:** `Started MindGuardApplication in X seconds`

### 2. Start Frontend (2 minutes)
```bash
cd c:\JAVA\mind-guard\frontend

npm start -- --port 4201
```

**Wait for:** `Application bundle generation complete`

## 🔍 How It All Works

```
1. PATIENT CREATES JOURNAL ENTRY
   ↓
2. BACKEND RECEIVES REQUEST (with JWT token)
   ↓
3. AIANALYSISSERVICE.ANALYZEENTRY() CALLED
   ├─ PARALLEL EXECUTION:
   │  ├─ LLM CALL #1: Sentiment Analysis (500-800ms)
   │  ├─ LLM CALL #2: Distress Detection (400-700ms)
   │  └─ Check if Distress > 5.0
   │
   └─ IF CRITICAL:
      └─ LLM CALL #3: AI Analysis (1000-2000ms)
   ↓
4. RESULTS STORED IN DATABASE
   ├─ Journal entry
   ├─ Sentiment score
   ├─ Distress level
   └─ Alert (if critical)
   ↓
5. THERAPIST SEES ALERT ON DASHBOARD
   ├─ With AI-generated insights
   ├─ With sentiment scores
   └─ With action buttons
```

## 📝 Important Notes

### LLM Calls Are Async
- User doesn't wait for all 3 calls
- Happens in background
- Alert appears within 2-3 seconds

### Error Handling
- If Hugging Face API fails, gracefully degrades
- Alert is still created, just without AI analysis
- System remains functional

### Security
- All API calls require JWT authentication
- Passwords hashed with BCrypt
- CORS properly configured
- CSRF disabled for API (by design)

## 🐛 Troubleshooting

### Backend won't start
```bash
# Check if port 8081 is in use
lsof -i :8081
# Kill the process
kill -9 <PID>
# Try again
```

### Frontend shows error
- Clear cache: Ctrl+Shift+Delete
- Refresh: Ctrl+F5
- Check console: F12 → Console tab

### LLM calls not working
- Verify API key is set: `<YOUR_HUGGING_FACE_API_KEY>`
- Check backend logs for errors
- Verify internet connection

### Alert not appearing
- Wait 2-3 seconds (LLM calls take time)
- Refresh page
- Check if distress level > 5.0
